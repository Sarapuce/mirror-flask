---
name: commit
description: Perform a good commit. Use it each time you need to commit changes
---

When doing a commit:

- Use the conventional commits format: `<type>: <short description>` (e.g. `feat: add port argument`, `fix: correct default value`, `docs: update README`)
- Keep the message on a single line, as short and clear as possible
- Focus on *what* changed and *why*, not *how*
- Never add "Co-authored-by" or any Claude attribution
